Ology is a browser-based audio-reactive kaleidoscope for meditation, neuroplasticity, and live performance. This guide covers every control, the full drone library, OBS setup, and ready-to-use session recipes.

01 What Ology is

Ology renders an infinite, never-repeating fractal visual driven in real time by an internally synthesized audio engine. No sample files, no installation, no cloud services. Open the URL and the experience begins.

Three contexts

  • Therapeutic sessions. Grief processing, anxiety relief, focus recovery, and inner journeying. Visuals and entrainment drones are paired by emotional and neurological intent.
  • Neuroplasticity work. Structured sessions using binaural, isochronic, and monaural entrainment targeting specific EEG frequency bands correlated with learning, memory, and peak cognition.
  • Live performance. DJ visuals, podcast backdrops, and OBS broadcast content. The two-page architecture keeps audio and video cleanly separated for the OBS bus.

Design principle

Calm by default. Intense on demand.

Therapeutic presets bias toward slow, smooth, low-contrast motion. Performance presets unlock chaos, high-speed transitions, and visual density — always within the built-in strobe limiter that is on by default.

💡
No auto-play Audio never starts on its own. Press Play in the Audio tab to begin.

02 Interface

Ology has two form factors — the desktop Stage (web.html) and the Mobile App. Both share the same PREVIEW / PROGRAM model and identical controls. This section covers the desktop Stage in detail; the Mobile App layout follows below.

Desktop Stage layout

The Stage is divided into four permanent zones that are always visible. Nothing is hidden behind a fullscreen toggle or overlay — the operator sees everything at once.

web.html · Desktop Stage — four zones annotated
A — Topbar: logo + engine button
B — Tab sidebar: Visuals · Audio · Output · Presets
C — Canvas: PREVIEW (blue) + PROGRAM (red)
D — Control panel: active tab content + GO

Zone A — Topbar

The topbar runs across the full width of the page and is always visible. It contains the Ology brand logo on the left and the ❚❚ Running engine button on the right. The engine button pauses and resumes the visual render loop — it does not affect audio. The button shows Running in teal when active and Paused when stopped.

Zone B — Tab sidebar

The left sidebar holds the four navigation tabs. The active tab is highlighted in purple. Clicking a tab switches the control panel (Zone D) to that tab's content. The canvas (Zone C) is unaffected by tab switching — it keeps rendering regardless of which tab is open.

👁 Visuals

Theme selector and all visual parameter sliders. Changes go to PREVIEW only — press GO to commit to PROGRAM.

🔊 Audio

Transport, drone preset, entrainment mode, ambient bed, blend, brain-tickle, slowness, reactivity routing, and master volume.

📤 Output

Aspect ratio selector, param overlay toggle, and the Open output window button for OBS.

💾 Presets

Save, load, favorite, delete, export, and import complete session snapshots.

Zone C — Canvas

The canvas zone holds two monitors side by side — PREVIEW on the left and PROGRAM on the right. This is where you see the kaleidoscope running in real time.

Canvas monitors — PREVIEW staging · PROGRAM live

PREVIEW (blue border) is your private staging area. All slider and theme changes go here first. Your audience never sees PREVIEW — only you do. Use it to audition a new look without disturbing a live session.

When you are happy with what you see in PREVIEW, press GO to push it to PROGRAM instantly.

PROGRAM (red border · LIVE) is what your audience sees — it is the canvas that syncs to the output window and OBS. It only changes when you press GO.

Audio parameter changes (drone, mode, bed, volume) bypass this model and take effect immediately in both monitors without pressing GO.

Zone D — Control panel

The right panel is the active tab's control surface. It changes completely depending on which tab is selected in Zone B. All visual controls in this panel write to PREVIEW only. The panel always ends with the GO button which commits the current PREVIEW state to PROGRAM.

GO

GO is the only action that changes what your audience sees. Every visual change you make — theme, speed, chaos, bloom, segments — stages silently in PREVIEW until you press GO. Audio changes (drone, mode, bed, volume) take effect immediately and do not require GO.

03 Canvas

Ology uses a two-bus model borrowed from broadcast switching: every change is staged in PREVIEW before it goes live in PROGRAM. Nothing happens to the live output until you deliberately commit it.

GO

Pressing GO copies all current PREVIEW settings — theme, speed, chaos, zoom, segments, bloom, smooth — into PROGRAM and makes them live instantly. Audio changes are live immediately without GO.

PREVIEW (blue)

  • Safe staging area — nothing visible to your audience or OBS yet.
  • All slider and theme changes go here first.
  • Loading a preset always loads into PREVIEW, never directly into PROGRAM.
  • You can audition a visual change without interrupting a live session.

PROGRAM (red · LIVE)

  • The live output — what appears on the OBS canvas and any connected output.
  • Committed by pressing GO.
  • Audio parameters (drone, mode, bed, volume) take effect immediately and bypass the GO step.
  • The param overlay on the Output Window always reflects PROGRAM state.
ℹ️
Audio vs Visual — two different commit rules Visual parameters stage into PREVIEW and require GO. Audio parameters (drone selection, entrainment mode, carrier Hz, beat Hz, ambient bed, blend, volume) take effect immediately when you change them — there is no PREVIEW step for audio.

04 Visuals Tab

All visual parameters update PREVIEW only. Press GO to make any change live.

Theme selector

The Preview Theme dropdown selects which of the seven visual themes to load into PREVIEW. The default theme on load is Intergalactic Beings for both PREVIEW and PROGRAM. Each theme has a distinct shader, colour palette, and geometry character. See Themes for the full list.

Motion controls

Intensity 0 – 1

Fractal depth. Higher values produce more detailed, recursive geometry. Lower values produce softer, more dissolving forms. Default 0.50.

Speed 0.1× – 3×

Animation rate. Therapeutic presets default to 0.6×–0.8×. Performance presets may run at 1.5× or higher. Default 0.80.

Chaos 0 – 1

Noise modulation amplitude. High chaos = organic, lava-lamp fluidity. Low chaos = crystalline, geometric precision. Default 0.50.

Rotation 0 – 1

Slow rotational drift of the entire canvas. At 0, the image holds still. Default 0.50.

Zoom near – far

Camera distance. far opens a wide-field view of the fractal surface. Closer values push into the interior geometry.

Segments 2 – 24

Kaleidoscope symmetry count. 2 = mirror. 4 = four-fold. 8 = octagonal. Even numbers produce radial symmetry; odd numbers produce spiral offset.

Bloom 0 – 1

Luminance blooming around bright areas. Increases the ethereal quality of the visual. Default 1.00.

Smooth 0 – 1

Edge sharpness. At 1.0, edges are soft and diffuse. At 0, edges are crisp and defined. Default 0.82.

Symmetry type

The three Symmetry buttons select the fold type applied before the segment count:

  • Radial — rotational symmetry around the centre. Clean and mandala-like.
  • Mirror — bilateral reflection. Creates left/right symmetry.
  • Kaleidoscope — the full domain-folding kaleidoscope. The most visually complex mode. Default.

05 Audio Tab

All audio in Ology is generated algorithmically — no sample files, no recordings. The audio engine synthesizes carrier tones, entrainment beats, and six ambient texture beds entirely in the browser.

Transport

Stop clears and rebuilds the audio engine, resetting all oscillators cleanly. Play starts the drone and ambient bed. Audio never auto-starts on page load.

Drone generator

The Drone preset dropdown selects one of 13 presets. The display shows the name, carrier frequency, beat frequency, and band — for example: Theta · Intuition · 136.1 Hz / 6 Hz. See the Drone Library for all 13 presets.

Carrier Hz

The audible base tone — what you actually hear. Adjusting the carrier changes the pitch of the drone without changing the entrainment target.

Beat Hz

The entrainment target frequency — the difference between L/R channels (binaural), the pulse rate (isochronic), or the modulation rate (monaural). The band label updates live as you move this slider.

Entrainment mode

Binaural
Headphones required

A different frequency reaches each ear. The brain perceives a third tone equal to the difference. Requires true stereo separation. Most documented in EEG research.

🎧
Headphone advisory

Binaural requires stereo headphones for the psychoacoustic effect. On speakers or in OBS the two tones blend and the effect is not delivered. Use Isochronic or Monaural for speaker or broadcast use.

Isochronic
Speakers OK

A single tone pulsed on and off at the entrainment rate. Sharp onset and offset. No headphones required. Preferred for stage speakers, OBS, and livestreams.

⚡
Flashing light risk

Isochronic pulses at 1–40 Hz. With audio reactivity active this can produce visual luminance pulses at the same rate. Users with photosensitive epilepsy should use Binaural or Monaural instead. Inform all participants.

Monaural
Speakers OK

Two tones mixed in a single channel, creating physical amplitude modulation. A real acoustic beat — no headphones required. Strong entrainment stimulus.

🔊
Speaker friendly

Monaural beats are created by mixing two tones before they reach the speakers, producing a physical beat in the air. Works with any playback system — speakers, OBS, broadcast, or headphones.

Texture

Brain-tickle 0 – 1

Subtle harmonic overtone pulses layered over the drone. Adds texture and aliveness to the carrier tone. Default 0.20.

Slowness 0 – 1

An LFO that introduces a very slow undulation to the drone's timbre — a breathing quality. Higher values = slower, deeper breath cycle. Default 0.60.

Ambient bed

Six generative ambient textures synthesized entirely from oscillators and filters — no recordings used. Select one from the Ambient Bed dropdown.

Womb

Low, warm, resonant rumble. Enveloping and primal.

Waterfall

Broadband white noise shaped into flowing water.

Jungle

Dense mid-frequency texture with subtle organic modulation.

Forest

Open, airy. Sparse texture with high-frequency shimmer.

Whales

Deep low-frequency long-cycle tones. Slow and vast.

Ocean

Wave-like low-frequency swells with broadband surface texture.

The Drone ↔ Bed blend dial controls the equal-power crossfade. All the way left = pure drone. Centre = equal mix. All the way right = pure ambient bed.

Loop & Master

Crossfade

Seamless looping. The drone cycles without audible restart.

Evolving

Slowly mutates the drone's timbre over time. No two moments identical.

One-shot

Plays the drone through once and stops.

Master

Post-crossfade output volume. Default 0.80.

Reactivity — Band → Visual

This matrix routes live audio frequency analysis into the visual engine in real time. The drone audio is analysed into four bands every frame; this panel maps each band's energy to a visual parameter.

Sub → Bloom

Sub-bass energy (below ~80 Hz) drives bloom intensity. Heavy bass = more glow.

Low → Zoom

Low-mid energy (~80–400 Hz) drives camera distance. Bass pulses pull and push the view.

Mid → Chaos

Mid-frequency energy (~400 Hz–2 kHz) drives the chaos noise modulation.

High → Color

High-frequency energy (2 kHz+) drives colour palette shift.

💡
Reactivity only works while audio is playing All four routes are gated on audio playback. If the transport is stopped, the visual falls back to its static parameter values and audio reactivity is suspended.

06 Output Tab

The Output tab controls the Output Window, the param overlay, and wireless phone pairing.

Why two separate windows?

Ology uses two separate windows. The Desktop App is the control surface — it generates all audio and runs the operator's UI. The Output Window is a pure video canvas — no audio, no UI chrome, just the full-screen kaleidoscope.

ℹ️
Why audio and video are separate OBS sources If the Output Window also generated audio, OBS would receive two identical drone signals causing phasing artifacts and destroying binaural left/right channel separation. Keeping audio exclusively in the Desktop App preserves clean, phase-correct stereo on the OBS recording bus.

Aspect ratio

Select the output aspect ratio before opening the Output Window. The default on load is 16:9. The canvas viewer adapts to each selection:

  • 16:9 · 1:1 · 4:3 — canvas fits side to side within the monitor viewer.
  • 9:16 — canvas fits top to bottom, displaying the full portrait frame without cropping.

Param overlay

Toggle On / Off to show or hide a live parameter readout in the top-left corner of the Output Window. Displays theme name, drone, carrier/beat Hz, mode, bed, brain-tickle and slowness values. Colour is theme-derived and updates each frame. The overlay remains visible even when the Desktop App engine is paused — confirming the Output Window canvas is still live.

13 OBS Setup

Ology sends video and audio to OBS as two separate sources. Video comes from the output popup window captured via Window Capture. Audio is captured directly from the browser running the control page. This separation preserves binaural stereo integrity on the OBS recording bus.

Step 1 — Open the output window

  1. In Ology, open the Output tab in the right panel.
  2. Select your aspect ratio (16:9 · 9:16 · 1:1 · 4:3). Default is 16:9.
  3. Click ⛶ Open output window. A popup opens and immediately requests fullscreen.
  4. When your browser prompts "Allow this page to go fullscreen?" — click Allow. This only happens once.
  5. The output canvas fills your entire screen with no address bar, no tabs, no OS chrome.
💡
Press Escape to exit fullscreen The window stays open when you exit fullscreen — just the chrome reappears. Click the button again to return to fullscreen. Keep this window open throughout your session.

Step 2 — Add the video source in OBS

  1. In OBS Studio, go to the Sources panel and click +.
  2. Choose Window Capture.
  3. Name it Ology Video and click OK.
  4. In the Window dropdown, select the Ology output window — it appears as something like Opera - Ology.
  5. Check Client Area to crop out the thin OS window border.
  6. Click OK. The Ology canvas should fill the OBS source.
ℹ️
Window Capture method Use Windows Graphics Capture as the capture method in OBS for the cleanest result on Windows. If the source appears black, try switching to BitBlt in the Window Capture properties.

Step 3 — Add the audio source in OBS

  1. In the Sources panel, click + again.
  2. Choose Application Audio Capture (Windows) or Desktop Audio (Mac).
  3. Set it to capture audio from the browser window running the Ology control page.
  4. Verify audio levels appear in the OBS audio mixer — you should see the drone waveform moving when audio is playing.
⚠️
Keep the control page open The Ology control page (localhost:8080) must stay open and running throughout the session. It is the sole audio source and writes the live visual state to the output window every frame via localStorage. Closing it stops both audio and visual updates.

The param overlay

Toggle On in the Output tab to show a live parameter readout in the top-left corner of the output window. Displays theme name, drone, carrier/beat Hz, entrainment mode, ambient bed, and timing values. Font scales with the output window size. The overlay stays visible even when the Stage engine is paused — confirming the output canvas is still live.

14 Safety

⚡
Isochronic mode — flashing light risk Isochronic entrainment pulses audio on and off at the beat frequency (1–40 Hz). With audio reactivity active this can produce corresponding visual luminance pulses at the same rate. Users with photosensitive epilepsy or light sensitivity should use Binaural or Monaural mode instead. Inform all participants before sessions that involve Isochronic mode.
🔆
Strobe limiter — on by default The maximum flash rate is limited by the Smooth parameter and the drone's audio reactivity envelope. Keep Smooth above 0.7 in therapeutic contexts. Users with a history of photosensitive epilepsy should consult a physician before use.
🎧
Headphones for Binaural only Binaural mode requires stereo headphones. On speakers, the two tones blend and the intended psychoacoustic effect is not delivered. Use Isochronic or Monaural for speaker or broadcast use.
🧘
Safe environment Use Ology in a comfortable, quiet setting. Do not use while driving, operating machinery, or in any situation requiring active attention.
🌿
Spiritual frequencies The Solfeggio frequencies (396–963 Hz) and chakra attributions are spiritual tradition claims, not clinically established findings. Presented as carriers for meditation intention only.
💡
Not a medical device Ology is not intended to diagnose, treat, cure, or prevent any condition. All frequency-to-state mappings are experiential meditation tools based on tradition and preliminary research, not clinical protocols. Consult a qualified professional for therapeutic treatment.
